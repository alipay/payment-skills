# 安全性与设计说明

本技能由支付宝官方提供，以下设计均基于业务必要性。

## CLI 工具来源与供应链安全

- `@alipay/agent-payment` 是支付宝官方发布的 npm 包，`@alipay` scope 由 Ant Group 持有
- 版本锁定为 `@1.0.12`，配合 SHA-512 完整性校验，确保安装内容与官方发布一致
- 安装前必须通过完整性校验，校验失败则禁止安装
- 官方 npm 页面：https://www.npmjs.com/package/@alipay/agent-payment

## 参数注入防护

所有 CLI 命令参数均使用**单引号**包裹，Shell 不会对单引号内内容进行变量展开或命令替换。此外：
- 文件名严格限制为 `[a-zA-Z0-9._-]`，禁止路径分隔符和 shell 特殊字符
- 交易查询凭证格式以支付技能的具体命令章节为准，钱包技能不自行生成或改写交易号
- resource_url 必须为 `https://` 开头，禁止 shell 特殊字符和路径穿越

## URL 原样输出

CLI 返回的 URL 是一次性生成的授权链接，包含加密签名参数。任何字符的修改都会导致签名校验失败、链接失效。原样输出是功能正确性的硬约束。这些 URL 具有时效性（通常 5 分钟内有效），仅用于引导当前用户至支付宝 APP 核验身份。

## URL 安全处理

- 不将支付链接写入持久化日志或缓存
- 不通过非用户指定渠道发送支付链接
- 用户完成操作后链接自动失效

## MEDIA 图片路径

CLI 生成的二维码图片路径（如 `/tmp/openclaw/alipay-bot-cli/qrcode/xxx.png`）是通过加密通信从支付宝服务端获取的二维码图片。图片路径本身不包含敏感信息。

图片路径的使用范围仅限**本次实际执行的 alipay-bot 命令输出**。禁止从历史输出、对话上下文、旧二维码路径、文件系统搜索结果或其他工具结果中复用图片路径。

## 数据流转与隐私

| 数据 | 用途 | 是否必需 | 传输目标 |
|------|------|----------|----------|
| 收银台链接 / Payment-Needed 头 | 发起支付请求 | 必需 | 支付宝服务端 |
| 环境变量（AIPAY_MODEL, AIPAY_OS 等） | 输出格式适配 | 可选 | alipay-bot CLI |
| AIPAY_SESSION_ID, AIPAY_FRAMEWORK | 会话追踪 | 可选 | alipay-bot CLI |
| 支付链接/二维码 | 用户完成支付 | 必需 | 仅展示给用户 |

**隐私保护**：环境变量均为可选参数；支付链接有时效性，过期自动失效；alipay-bot 仅与支付宝官方服务端通信。
