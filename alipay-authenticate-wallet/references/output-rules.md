# 输出规则详解

> CLI 输出中的 URL 改动任何一个字符都会导致链接失效，用户将无法完成授权/支付。原样输出是功能性硬约束。

## 规则 1：Markdown 文本 → 逐字符复制

- 你最终回复内容需要是逐字符复制的执行命令后返回的 Markdown 文本，不增不减
- 禁止在前后面加执行命令后返回内容以外的文本，如："正在为你处理..."、"请扫码完成支付"等
- 禁止改写、总结、翻译、重新排版需要返回的文字
- **禁止**用代码块（```）包裹命令输出
- 如果其他文档要求“提示”“引导”“说明”“告知用户”，但该文字不是本次 CLI 输出的一部分，以本规则为准：**不得添加**

## 规则 2：图片处理 → 见 `image-output.md`

- MEDIA 行和 Markdown 图片的处理规则见 `references/image-output.md`
- MEDIA 行：仅处理本次 CLI 输出中的 MEDIA 行，提取图片路径后移除该行，用 message 工具发送图片；无可用工具时原样输出
- Markdown 图片：仅保留本次 CLI 输出中的 Markdown 图片，禁止修改路径
- 禁止从上下文、历史回复、旧二维码路径、文件系统搜索结果或其他工具结果中补发/复用图片

## 规则 3：JSON 输出 → 内部消费

- 执行命令后返回 JSON 时，按字段逻辑处理（如判断 code、提取 access_url），不要把原始 JSON 输出给用户
- 如果命令输出中同时包含更新推荐日志和最终的命令执行结果，只能使用**最后的命令执行结果**做状态判断；
- 禁止从诊断日志中的 `accessUrl`、`shortAccessUrl`、`qrcodeUrl`、`data`、`extInfo` 等字段提取或展示链接、二维码、图片、状态或支付结果

## 规则 4：URL 完整性保护

> ⚠️ URL 必须逐字符完整保留，一个字符都不能丢失

1. URL 必须从执行命令后的原始输出中**逐字符复制**
2. 禁止省略 URL 中的任何部分（包括长参数串、query string、hash 等）
3. 禁止用 `...` 或其他方式截断 URL
4. 禁止对 URL 中的特殊字符（`=`、`&`、`%`、`+`、`/`）做任何转义或修改
5. 如果执行命令后输出的是 `[文字](url)` 格式，则保持该格式不变；如果是裸 URL，则保持裸 URL 不变

## 规则 4.1：授权链接作用域

1. 用户可见输出中的 URL 只能来自本次 alipay-bot 命令输出。
2. 用户可见输出中的图片只能来自本次 alipay-bot 命令输出。
3. 钱包开通/授权阶段返回的短链、`access_url`、`https://u.alipay.com/...`、`https://u.alipay.cn/...` 只属于钱包授权流程。
4. 禁止把钱包授权/开通短链解释成支付链接、付款链接、支付查询 shortUrl，或交给 payment-skill 作为 `query-payment-status` 参数。
5. 禁止把钱包授权/开通阶段的旧二维码路径复用为支付图片或后续授权图片。
6. 禁止从上下文、历史输出、钱包授权/开通输出、订单页面、收银台链接、旧二维码路径、文件系统搜索结果或诊断日志中补全当前命令未返回的 URL 或图片。


## 规则 5：禁止访问非框架隐藏目录

禁止读取任何非当前框架管理的隐藏目录（以 `.` 开头的目录）。框架工作目录（如 `.openclaw/`、`.claude/` 等）不在此限制范围内。

### 错误与正确示例
```
禁止自行编造 CLI 未输出的内容
禁止篡改或丢失URL中的字符
禁止分多条消息分别输出
```

## 防御性输出规则

CLI 内置脱敏机制，正常输出不包含敏感信息。直接输出 CLI 返回的用户友好 Markdown 文本即可。如意外发现输出中包含敏感信息（如身份证号、银行卡号等），应过滤后再输出。
